# Introducao

A qualidade de software e fator critico para qualquer plataforma de comercio eletronico, pois impacta diretamente experiencia do cliente, conversao de vendas e confiabilidade operacional. No contexto do EBAC Shop, fluxos como login, carrinho e promocoes por cupom exigem validacao sistematica para reduzir falhas em producao.

Este trabalho de conclusao foi desenvolvido com abordagem de Quality Engineering ponta a ponta, considerando o ciclo completo de testes: planejamento, definicao de criterios, desenho de casos, automacao, execucao e evidencia. O projeto foi estruturado para reproduzir um contexto real de time agil, com rastreabilidade entre requisitos e testes e incorporacao de requisitos de qualidade segundo o modelo SQuaRE.

Como resultado esperado, a entrega consolida uma estrategia hibrida (manual e automatizada), com cobertura multi-plataforma (web, API e mobile Android), validacao de contrato, testes de performance e pipeline de integracao continua. Alem disso, adota-se um processo formal de avaliacao de qualidade (ISO/IEC 25010, 25023 e 25040) com scorecard e criterios de gate para suporte a decisao de liberacao.
